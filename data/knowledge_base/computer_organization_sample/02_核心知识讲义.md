# 计算机组成原理核心知识讲义

## 1. 冯诺依曼结构

冯诺依曼结构是现代通用计算机的经典结构模型，其核心思想是“存储程序”和“程序控制”。它通常由运算器、控制器、存储器、输入设备和输出设备五大部分组成。

冯诺依曼结构的关键特征包括：

1. 指令和数据以二进制形式统一存放在存储器中。
2. 程序按地址顺序执行，遇到转移指令时再改变执行顺序。
3. 控制器负责取指、译码和发出控制信号，协调各部件工作。

冯诺依曼结构的优点是结构清晰、通用性强，但也存在“存储墙”和“总线瓶颈”等问题，因此现代系统通常会引入 Cache、流水线和并行处理等优化机制。

## 2. 指令周期

一条指令从开始取出到执行结束，通常要经历一个完整的指令周期。典型阶段包括：

1. 取指：根据程序计数器 PC 的内容，从主存中取出指令。
2. 译码：分析操作码和地址码，确定要执行的操作及操作数来源。
3. 执行：由运算器、寄存器或输入输出部件完成对应操作。
4. 写回：如果有结果产生，则写回寄存器或主存。

在不同结构中，指令周期可能进一步细分成机器周期和时钟周期。理解这几个层次的关系，是分析性能和控制流程的基础。

## 3. 流水线技术

流水线技术的核心思想是把一条指令的执行过程划分为多个阶段，让多条指令在不同阶段并行推进，从而提升吞吐率。

例如，一个经典五级流水线可以分为：

1. IF：取指
2. ID：译码
3. EX：执行
4. MEM：访存
5. WB：写回

流水线并不一定降低单条指令的延迟，但通常能够提高单位时间内完成指令的数量。流水线常见问题包括：

1. 结构冒险：多个阶段同时竞争同一硬件资源。
2. 数据冒险：后一条指令依赖前一条尚未写回的结果。
3. 控制冒险：分支转移导致后续指令流不确定。

常见解决办法包括资源复制、前递技术、暂停等待和分支预测。

## 4. Cache 机制

Cache 是位于 CPU 与主存之间的一种高速小容量存储器，用于缓解 CPU 与主存速度差距过大的问题。

Cache 的设计依赖程序访问局部性原理：

1. 时间局部性：最近访问过的数据，未来还可能被再次访问。
2. 空间局部性：访问某个地址后，附近地址也可能很快被访问。

Cache 的主要作用是降低平均访存时间。评价 Cache 效果时，通常关注：

1. 命中率
2. 命中时间
3. 失效率
4. 失效代价

如果命中率较高，CPU 大部分访存请求都能直接由 Cache 满足，系统整体性能就会得到明显提升。

## 5. 中断机制

中断是指 CPU 在执行当前程序过程中，响应外部或内部事件请求，转去执行相应服务程序的机制。

中断机制的意义主要体现在：

1. 提高输入输出效率，不必一直用程序查询等待设备。
2. 增强系统对异常事件的响应能力。
3. 支持多任务环境中的资源协调与控制转移。

中断处理的一般过程包括：

1. 中断请求产生
2. 中断判优
3. 保存断点和现场
4. 转入中断服务程序
5. 恢复现场并返回原程序

中断方式相较于程序查询方式更加高效，但硬件和控制逻辑也更复杂。

## 6. 学习本章时的关注重点

1. 冯诺依曼结构要关注“为什么程序和数据统一存储”。
2. 指令周期要关注“控制器如何驱动一条指令完成执行”。
3. 流水线要关注“为什么提高吞吐率却会带来冒险”。
4. Cache 要关注“局部性原理与平均访存时间”。
5. 中断要关注“效率提升”和“控制流程变化”。
