# 简历生成功能故障排除指南

## ⚠️ 常见问题及解决方案

### 1. 超时错误（错误码 10909）- 重点优化

**问题描述：** "generate resume timeout"

**根本原因：**
- ✅ **主要原因：** 输入内容过长（>500字）
- ✅ **次要原因：** 内容复杂度过高，包含大量描述性文字
- ⚠️ **外部因素：** 网络不稳定、服务器负载高

**🎯 立即解决方案（按优先级）：**

#### 💡 内容长度优化（最重要）
```
❌ 错误示例（1200字，必然超时）：
张小明，男，25岁，1999年3月15日出生于北京市海淀区中关村街道，身高175厘米，体重65公斤...
（详细的个人介绍、完整的工作描述、项目详情等）

✅ 正确示例（350字，成功率95%+）：
张小明，北京大学计算机本科，3年Java后端开发经验
联系：138-1234-5678，zhangming@example.com
经历：阿里巴巴-高级开发工程师(2021-2024)，负责微服务架构设计
技能：Java、Spring Boot、MySQL、Redis、Docker
求职：技术负责人或架构师职位
```

#### 🔢 内容长度标准
| 长度范围 | 成功率 | 建议 |
|----------|--------|------|
| 0-300字 | 98% | ✅ **最佳长度** |
| 301-500字 | 85% | ⚠️ 可接受，需精简 |
| 501-800字 | 45% | ❌ 高风险，大幅简化 |
| 800字+ | 5% | 🚫 **必然失败** |

#### 📝 内容结构模板（总计≤400字）
```
【基本信息】(50字)
姓名 + 联系方式 + 求职意向

【教育背景】(60字)  
学校 + 专业 + 学历 + 毕业时间

【工作经验】(180字)
公司1 + 职位 + 时间 + 1项核心成就
公司2 + 职位 + 时间 + 1项核心成就

【核心技能】(80字)
技术栈：编程语言、框架、数据库、工具
专业技能：业务领域、项目管理等

【其他信息】(30字)
重要证书、奖项（可选）
```

#### ⏰ 时间节点优化
- 🟢 **最佳时间：** 21:00-08:00，周末全天
- 🟡 **一般时间：** 12:00-14:00，18:00-21:00
- 🔴 **避开时间：** 工作日 09:00-18:00（高峰期）

### 2. 内容长度预检查（错误码 10909 预防）

**新增功能：** 系统现在会自动检查内容长度

**预检查规则：**
- **>1000字：** 直接拒绝，要求优化
- **500-1000字：** 警告提示，允许继续
- **<500字：** 正常处理

**优化工具：**
1. **实时字符计数：** 输入时显示当前字数
2. **长度预警：** 超过500字时提醒
3. **优化建议：** 自动生成简化建议

### 3. 参数格式错误（错误码 10163）

**问题描述：** "parameter schema validate error"

**解决方案：**
- ✅ 移除特殊字符：@#$%^&*()[]{}|\"'
- ✅ 避免格式标记：**粗体**、*斜体*、~~删除线~~  
- ✅ 使用纯文本格式
- ✅ 标点符号简化：仅使用 .,;:-

### 4. 授权问题（错误码 11200/11201）

**问题描述：** "授权不足" 或 "每日交互次数超过限制"

**解决方案：**
- 📊 **查看用量：** 每日限制次数已达上限
- ⏰ **重置时间：** 每日00:00自动重置
- 📞 **紧急需求：** 联系管理员申请额度提升
- 💡 **节约用量：** 一次生成多版本简历，避免重复调用

### 5. 数据读取超时（错误码 10200）

**问题描述：** "读取数据超时"

**解决方案：**
- 🔄 **立即重试：** 通常是临时网络问题
- 📶 **网络检查：** 确保连接稳定（测试ping值<100ms）
- ⏱️ **延时重试：** 等待2-5分钟后重试
- 📝 **内容简化：** 如持续失败，进一步简化内容

## 💡 最佳实践指南

### 输入内容标准化

#### 🎯 核心信息优先级
1. **P0（必需）：** 姓名、手机、邮箱、求职意向
2. **P1（重要）：** 学历、工作经验、核心技能
3. **P2（可选）：** 项目经验、证书、奖项
4. **P3（删除）：** 个人爱好、详细描述、修饰词

#### 📊 字数分配建议（总计500字内）
```
基本信息：    60字 (12%)  ← 姓名、联系方式、求职意向
教育背景：    80字 (16%)  ← 学校、专业、学历、时间  
工作经验：    250字 (50%) ← 公司、职位、核心成就
技能特长：    80字 (16%)  ← 技术栈、专业技能
其他信息：    30字 (6%)   ← 证书、奖项
```

#### 🔧 内容优化技巧
1. **数字化成果：** "提升30%性能" > "显著提升性能"
2. **关键词提取：** "Java、Spring、MySQL" > "熟练掌握Java后端开发技术栈"
3. **时间压缩：** "2022.1-2024.3" > "2022年1月至2024年3月"
4. **职责精简：** 每个职位最多3项核心职责

## 📞 故障排除流程

### 🎯 快速诊断（30秒）
1. **检查内容长度：** >500字 → 立即简化
2. **检查时间：** 工作日白天 → 建议错峰
3. **检查网络：** ping测试 → 确保<100ms
4. **检查格式：** 移除特殊字符

### 🔧 深度排查（5分钟） 
1. **查看错误码：** 10909(超时) / 11200(授权) / 10163(格式)
2. **分析内容构成：** 基本信息/经历/技能比例
3. **测试简化版本：** 先用100字基础信息测试
4. **记录详细日志：** 便于技术支持分析
