# A3 测试说明书

## 1. 测试目标

本说明书用于验证"学智画像"在比赛要求下的核心功能可用性、交互完整性与演示稳定性，重点关注：

1. 核心功能是否可运行
2. 多智能体协同链路是否完整
3. 关键页面是否具有答辩展示价值
4. 启动与部署流程是否稳定

## 2. 测试范围

### 2.1 账号与权限

1. 演示账号自动补齐与密码校正
2. 学生、教师、管理员登录跳转
3. 管理员在教师工作台上的增强入口

### 2.2 智能问答与多智能体

1. 基础问答模式可访问
2. 多智能体模式可访问
3. 画像构建、资源生成、学习规划、评估入口可见
4. 智能问答页面赛题能力面板可见

### 2.3 能力测试

1. 普通高校专业题组可正常切换
2. 民航特色题组可正常切换
3. 逐题答题流程正常
4. 提交后可返回分数、难度分析、逐题解析

### 2.4 学习计划与学习报告

1. 学习计划页面可访问
2. 深度学习计划接口可返回有效数据
3. 学习报告页面可访问
4. 测评 -> 报告 -> 计划的导航链路可见

### 2.5 知识库与 RAG 验证

1. 课程样例文件可成功上传并建立索引
2. 知识库状态接口可返回文档数和分块数
3. 检索接口可返回相关材料片段
4. RAG 问答接口可返回引用回答、来源信息与置信度字段

### 2.6 工程化与部署

1. `start.bat` 可用于 Windows 双击启动
2. Docker 与 WSGI 文件存在

1. `python -m compileall app`
2. Flask `test_client()` 页面与接口冒烟验证

## 5. 典型测试场景

### 场景 1：学生能力测试闭环

1. 学生登录
2. 进入能力测试
3. 选择普通高校或民航特色方向
4. 完成答题
5. 查看总分、难度分布、逐题解析
6. 进入学习报告或学习计划页面继续学习

### 场景 2：知识库问答与资源生成

1. 登录智能问答页
2. 切换到多智能体模式
3. 构建学习画像
4. 上传课程样例包中的知识库材料
5. 发起知识问答或资源生成请求
6. 观察引用回答、资源入口与能力面板

### 场景 3：答辩演示

1. 首页展示项目入口
2. 能力测试展示专业化测评
3. 智能问答展示多智能体协同
4. 学习计划展示个性化路径
5. 学习报告总结学习成果
