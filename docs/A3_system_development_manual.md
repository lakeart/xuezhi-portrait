# A3 系统开发说明书

## 1. 项目概述

"学智画像"是一个面向高校大学生的个性化学习多智能体系统，围绕比赛中的五类核心能力展开：

1. 对话式学习画像构建
2. 多智能体协同资源生成
3. 个性化学习路径规划与资源推送
4. 智能辅导与即时答疑
5. 学习效果评估与动态优化

系统以 Web 应用为主要承载形态，适用于课程教学辅助、自主学习辅导、知识库问答、资源生成与学习效果追踪等高校场景。

## 2. 需求分析

### 2.1 用户痛点

1. 高校学生面对课程内容多、节奏快、资源杂，难以快速找到适合自己的学习材料。
2. 传统统一化教学难以兼顾学生基础差异与学习偏好差异。
3. 学习计划往往缺少动态调整机制，不能随着学习效果变化而变化。
4. 答疑系统常停留在单轮问答，缺少画像、知识库和资源生成联动。
5. 学习效果评估通常停留在总分层面，难以形成针对性的改进建议。

### 2.2 赛题结合点

1. 通过自然语言对话采集学生专业、目标、短板和偏好，构建动态画像。
2. 通过多智能体分工协作生成多模态学习资源。
3. 通过学习路径智能体输出个性化、可调整的学习计划。
4. 通过知识库 RAG 与智能问答降低幻觉风险，增强教学场景可靠性。
5. 通过能力测试、学习报告和计划回流形成闭环优化。

## 3. 总体架构

### 3.1 技术架构

系统采用 Flask 为核心后端框架，前端以服务端模板渲染为主，结合 JavaScript 增强交互体验。主要结构如下：

1. 表现层：登录、首页、智能问答、能力测试、学习报告、学习计划等页面
2. 路由层：负责业务接口与页面调度
3. 智能体层：负责画像、资源生成、学习规划、问答、评估、知识库检索
4. 数据层：用户、题目、答题记录、学习画像、学习资源、学习路径、知识库文档与分块

### 3.2 多智能体角色划分

1. 画像智能体：根据自然语言输入抽取学生特征，构建动态画像
2. 资源生成智能体：输出课程讲义、题库、思维导图、PPT、视频等多模态资源
3. 学习规划智能体：根据画像和掌握度生成阶段化学习路径
4. 辅导智能体：负责问答、解释、知识点讲解
5. 评估智能体：对学习过程与测评结果进行分析，输出学习建议
6. 知识库智能体：负责文档上传、解析、索引、检索与引用回答
7. 协调智能体：协调多角色工作流，形成完整闭环

## 4. 核心功能说明

### 4.1 对话式学习画像构建

1. 通过自然语言对话采集学习目标、课程背景、知识短板、兴趣方向等信息。
2. 系统已支持不少于 6 个维度的画像构建，当前实现扩展到 10 维。
3. 画像支持动态更新，并可用于后续资源生成与学习计划推荐。

### 4.2 多智能体协同资源生成

系统当前支持的资源类型包括：

1. 课程讲解文档
2. 知识点思维导图
3. 个性化题库
4. 拓展阅读材料
5. 代码类实操案例
6. PPT 资源
7. 教学视频/数字人视频任务
8. 知识卡片

这些资源由多个智能体协同完成，不同角色负责画像分析、知识检索、内容生成和结果展示。

### 4.3 个性化学习路径规划与资源推送

1. 支持按掌握度、目标、学习时段、重点知识点生成学习计划。
2. 提供日历视图、列表视图、学习策略视图、AI 洞察面板。
3. 支持风险提醒、重点知识点、资源推荐、任务节奏说明。
4. 支持从能力测试和学习报告结果回流到学习计划页面。

### 4.4 智能辅导

1. 支持基础知识问答与多智能体模式切换。
2. 支持知识库 RAG 优先回答，并给出引用依据。
3. 支持课程文档、PPT、思维导图、题库、视频等入口，体现资源与问答联动。

### 4.5 学习效果评估

1. 能力测试中心支持高校专业化测评。
2. 目前测试方向覆盖普通高校专业方向和民航特色专业方向。
3. 支持答题后输出总分、难度分布、优势、薄弱点、逐题解析和建议。
4. 学习报告页面用于展示趋势、能力雷达与个性化建议。

## 5. 知识库与反幻觉设计

### 5.1 当前实现

1. 支持 TXT、Markdown、CSV、JSON、DOCX、PDF 文档上传。
2. 完成文档解析、分块、索引、语义检索与引用回答闭环。
3. 回答优先依赖知识库证据，降低"无依据生成"风险。
4. 当前已补充"计算机组成原理"课程知识库样例包，可直接作为提交附件与演示输入。

### 5.2 反幻觉与内容安全机制

1. 知识库优先：当用户启用知识库问答时，系统优先调用用户上传材料进行检索与回答，而不是直接输出开放式生成结果。
2. 未命中拦截：若未检索到足够相关材料，系统返回"资料不足"提示，并给出继续上传课程讲义、课件或题库文档的建议。
3. 引用回答：RAG 返回结果会附带来源编号、文档标题、文件名和分块内容，便于追溯回答依据。
4. 教师材料优先：当多个来源存在表述差异时，优先建议以教师上传讲义、课程资料和实验文档为准。
5. 风险边界提示：对于高风险、缺材料或跨学科扩展问题，系统更适合给出"学习建议"和"材料引用"，而不是强结论式回答。
6. 后续增强方向：可继续补充敏感问题过滤、错误知识对抗样例、越权输入检测和人工复核机制说明。

### 5.3 样例包验证结果

本轮已使用 `data/knowledge_base/computer_organization_sample/` 中的 6 个课程样例文件完成真实上传与问答验证，结果包括：

1. 课程样例文件均可被解析并建立索引。
2. 知识库状态接口返回 6 份文档、7 个分块。
3. 对"什么是冯诺依曼结构""Cache 的主要作用是什么"等问题能够返回引用回答。
4. 验证结果与接口输出已整理到 `docs/evidence/api/`。

## 6. 主要技术与工程实现

1. 后端框架：Flask、Flask-SQLAlchemy、Flask-Login、Flask-WTF
2. 数据处理：pandas、numpy
3. 机器学习：scikit-learn
4. 图表展示：ECharts、Bootstrap
5. 文档解析：PyPDF2
6. HTTP 与外部接口：requests、websocket-client
7. 部署方式：Windows 启动、Docker、WSGI

## 7. 部署与运行

### 7.1 本地运行

1. 创建并激活虚拟环境
2. 安装依赖 `requirements.txt`
3. 启动 `run.py` 或双击 `start.bat`
4. 访问 `http://127.0.0.1:5000`

### 7.2 演示账号

1. `admin / admin123`

3. `student / student123`

3. `wsgi.py`
4. `start.bat`

## 8. 创新点总结

1. 将学习画像、知识库、资源生成、学习计划、测评反馈放入同一多智能体闭环。
2. 在能力测试中心中加入高校专业化方向与民航特色方向，增强场景化表达。
3. 将功能证据、完成度、下一步计划集中展示，适合答辩。
4. 在学习计划和学习报告页面突出"测评 -> 画像 -> 资源 -> 计划"的路径。
5. 通过真实课程样例包验证知识库链路，使"RAG 可信问答"不只停留在功能描述层面。
